# Rust & Logic 策划 Roadmap 与 TodoList

本文基于 [docs/game-design-guide.md](/D:/LabProjects/RustAndLogic/docs/game-design-guide.md) 制定，目标是把设计原则转成可执行的策划路线。

## 1. Roadmap 总览

建议用“阶段目标 + 编程主题 + 系统增量”的方式推进，而不是按题材点子堆料。

### Milestone 1：唤醒机器人

目标：

- 建立最基础的情感连接和动作反馈
- 让玩家确认“我写的代码真的让机器人动起来了”

玩家获得：

- `Move`
- `Turn`
- `PickUp`
- `Drop`

核心编程主题：

- 顺序执行
- 单步反馈

必须完成的内容：

- 初始废土场景
- 单机器人可见移动
- 资源拾取可见反馈
- 最简日志反馈

阶段完成标准：

- 玩家能写出第一段 3~5 行脚本
- 玩家能稳定感知“命令 -> 世界变化”

### Milestone 2：稳定拾荒循环

目标：

- 从“能动”进入“能自动工作”

玩家获得：

- 基础扫描能力
- 背包 / 电量判断
- 回家卸货
- 回家充电

核心编程主题：

- `while`
- `if`
- 简单优先级

必须完成的内容：

- Scrap / Wire / Cell / Plastic 基础资源层
- 背包容量
- 电量消耗
- 家 / 基地概念

阶段完成标准：

- 玩家能写出第一套稳定自动拾荒循环
- 机器人能“出门 -> 拾荒 -> 回家 -> 卸货 -> 继续”

### Milestone 3：临时据点与基础生产链

目标：

- 从自动拾荒升级为自动运转的小据点

玩家获得：

- 自动维修
- 自动充电
- 自动合成
- 基础模块升级

核心编程主题：

- 函数拆分
- 状态切换
- 基础生产链

必须完成的内容：

- 充电桩
- 维修台
- 合成台
- 第一批基础配方

阶段完成标准：

- 玩家脚本开始出现 `work()`、`return_home()`、`repair()` 这类结构
- 玩家感受到“不是机器人在工作，而是据点在运转”

### Milestone 4：危险垃圾区

目标：

- 制造第一次明确的“旧脚本失效”

玩家获得：

- 风险评估需求
- 回撤逻辑需求
- 容错需求

核心编程主题：

- 风险判断
- 滞回阈值
- 异常恢复

必须完成的内容：

- 辐射
- 耐久损耗
- 高价值稀有材料
- 失败后可恢复的损坏机制

阶段完成标准：

- 简单“看到资源就捡”的脚本会明显出问题
- 玩家必须写判断逻辑才能稳定盈利

### Milestone 5：黑市与动态经济

目标：

- 把固定阈值刷资源，升级为动态成本判断

玩家获得：

- 交易
- 蓝图
- 读取升级成本
- 自动补缺

核心编程主题：

- 动态目标
- 资源缺口调度
- 轻量经济 AI

必须完成的内容：

- 商人 / 黑市入口
- 价格和成本接口
- 自动升级链

阶段完成标准：

- 玩家不再只写“某资源低于 1000 就去刷”
- 玩家开始写“读取下一个目标成本，再决定刷什么”

### Milestone 6：角斗场试炼

目标：

- 建立第二条主玩法线：战斗 AI

玩家获得：

- 战斗读取接口
- 攻防 / 后撤 / 修理
- 基础状态机

核心编程主题：

- 状态机
- 敌人识别
- 目标优先级
- 冷却 / 热量管理

必须完成的内容：

- 多种敌人课题序列
- 战斗日志
- 战败后复盘入口

阶段完成标准：

- 角斗场成为“策略代码考试场”
- 玩家能明显感受到战斗脚本和拾荒脚本结构不同

### Milestone 7：迷宫与旧世界设施

目标：

- 把游戏推入空间推理与探索系统

玩家获得：

- 方向扫描
- 标记
- 记忆
- 路径回溯
- 地图构建

核心编程主题：

- 寻路
- 地图记忆
- 风险收益判断

必须完成的内容：

- 浅层迷宫
- 深层迷宫
- 限电 / 巡逻 / 锁门 / 陷阱
- 旧世界记忆碎片

阶段完成标准：

- 玩家从“遍历地图”升级到“构图找路”
- 剧情奖励与高级 API 解锁绑定

### Milestone 8：多机器人小队

目标：

- 从单体自动化升级为系统调度

玩家获得：

- 多机分工
- 共享状态
- 队列
- 广播 / 任务分配

核心编程主题：

- 并发
- 调度
- 资源竞争
- 全局优先级

必须完成的内容：

- 多机器人类型
- 多任务入口
- 共享仓储 / 共享维修 / 共享能源

阶段完成标准：

- 玩家开始写“总控系统”
- 多机器人不只是产量更高，而是行为更复杂

### Milestone 9：终局压力测试与机械聚落

目标：

- 把所有系统压成一个长期自我发展的自动化框架

玩家获得：

- 从零启动挑战
- 完全自动生产
- 完全自动战斗
- 完全自动探索
- 排行榜目标

核心编程主题：

- 综合自动化
- 全局资源调度
- 异常恢复
- 长周期稳定性

必须完成的内容：

- 终局混合关卡
- 多线压力同时存在
- 聚落建设结果可视化

阶段完成标准：

- 玩家追求的不再是通关，而是系统优化

## 2. 机制优先级建议

为了避免系统过早发散，策划上建议按以下优先级落：

### P0：必须先做扎实

- 单机器人移动 / 拾取 / 丢弃 / 回家
- 背包
- 电量
- 卸货
- 充电
- 维修
- 基础资源链
- 模块解锁 API

### P1：中期第一层扩展

- 危险区
- 黑市与工坊
- 动态升级成本
- 战斗 AI
- 战斗日志

### P2：中后期扩展

- 迷宫
- 路径记忆
- 旧世界剧情碎片
- 多机器人

### P3：终局系统

- 深层迷宫
- 竞技联赛
- 城市核心
- 聚落
- 排行榜

## 3. 关卡策划原则清单

每做一个新阶段，都先检查：

- 这个阶段教玩家的主编程问题是什么？
- 它会让哪一种旧脚本失效？
- 它新增了什么 API？
- 它新增了什么失败方式？
- 它的收益是否足以支付代码复杂度？
- 它是否和前一阶段形成了清晰对比？

如果有两项答不上来，这个阶段就不该进主线。

## 4. 数值策划原则清单

### 4.1 前期

- 升级成本低
- 单次资源反馈快
- 失败成本低
- 玩家能快速验证脚本收益

### 4.2 中期

- 用资源链制造隐性瓶颈
- 用危险区制造显性风险
- 用模块解锁表达能力
- 用动态成本替代固定阈值

### 4.3 后期

- 用混合目标制造调度压力
- 用多机器人制造并发竞争
- 用长周期挑战制造稳定性要求

### 4.4 统一要求

- 单阈值尽量改为滞回区间
- 流动资源尽量做节流逻辑
- 高级资源尽量依赖低级供给链
- 高收益区必须搭配高失败成本或高逻辑复杂度

## 7. 结论

从策划角度看，`Rust & Logic` 的 roadmap 不应该按“多几个区域、多几个敌人、多几个资源”来排。

它应该按：

- 新阶段引入什么新的编程问题
- 新阶段如何让旧代码失效
- 玩家如何因此被迫重构系统

来排。

只要 roadmap 始终围绕这条线推进，后续内容再扩，也不会散。
