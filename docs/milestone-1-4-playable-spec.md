# Rust & Logic Milestone 1~4 最小可玩策划稿

本文只覆盖前四个 Milestone，目标是定义一版真正可做、可测、可调整的前中期最小可玩方案。

范围原则：

- 不提前引入角斗场
- 不提前引入迷宫
- 不提前引入多机器人
- 先把“单机器人 -> 稳定拾荒 -> 小据点 -> 危险区重构”做扎实

## 1. 整体目标

在 Milestone 1~4 结束时，玩家应该已经能够：

1. 写基础动作脚本
2. 写稳定自动拾荒循环
3. 让据点自动卸货、回充、维修、合成
4. 在危险区中写出风险判断与回撤逻辑

换句话说，到了 M4 结束，玩家应该第一次感到：

> 我已经不是在操控一个角色，  
> 我是在维护一个能自己跑起来的小系统。

## 2. Milestone 1：唤醒机器人

### 2.1 体验目标

- 玩家捡到一个残破机器人
- 玩家写下最简单的命令
- 机器人确实动起来、转向、捡起东西

### 2.2 场景结构

- 一张极小地图
- 基地 / 出生点 1 个
- Scrap 若干
- 无危险
- 无时间压力

### 2.3 可用能力

- `Move`
- `Turn`
- `PickUp`
- `Drop`

### 2.4 限制

- 没有复杂扫描
- 没有危险区
- 没有复杂资源链
- 背包可先只有 1 格或 2 格

### 2.5 设计重点

- 动作反馈必须清晰
- 每个动作都要有演出反馈
- 日志要有轻微人格感

### 2.6 关卡任务建议

1. 让机器人向前走一格
2. 让机器人转向
3. 捡起 1 块废铁
4. 把废铁带回基地

### 2.7 完成标准

- 玩家能独立写出 3~5 行脚本完成搬运
- 玩家知道代码会直接影响地图世界

## 3. Milestone 2：稳定拾荒循环

### 3.1 体验目标

- 玩家第一次写出会自动重复工作的脚本

### 3.2 场景结构

- 中等小地图
- 基地 1 个
- 基础资源：`Scrap` `Wire` `Cell` `Plastic`
- 资源点可重复刷新或按批刷新

### 3.3 新增系统

- 背包容量
- 电量系统
- 卸货
- 回充
- 基础判断接口

### 3.4 玩家应学会的代码结构

- `while`
- `if`
- “如果前方有资源就捡，否则继续移动”
- “如果背包满了就回家”
- “如果电量低了就回家”

### 3.5 推荐关卡任务

1. 自动捡满一背包 Scrap
2. 自动回基地卸货
3. 自动回基地充电
4. 连续完成三轮拾荒而不报错

### 3.6 设计重点

- 这里不要引入太多惩罚
- 玩家第一次稳定循环成功，是前期最大爽点

### 3.7 完成标准

- 机器人能自动出门、拾取、回家、卸货、回充、继续
- 玩家开始感受到挂机价值

## 4. Milestone 3：临时据点

### 4.1 体验目标

- 从单机器人行为，升级为据点运转

### 4.2 场景结构

- 垃圾场边缘 + 小据点
- 据点内有：
  - 充电桩
  - 维修台
  - 零件箱
  - 合成台

### 4.3 新增系统

- 自动维修
- 自动充电
- 自动合成
- 第一条基础生产链

### 4.4 推荐基础配方

- `Scrap + Wire -> Motor`
- `Cell + Wire -> BatteryPack`
- `Scrap + Plastic -> Plate`

### 4.5 玩家应学会的代码结构

- `work()`
- `return_home()`
- `repair_if_needed()`
- `charge_if_needed()`
- `unload_then_continue()`

### 4.6 推荐关卡任务

1. 自动拾荒并在基地中持续堆出 Scrap
2. 自动回家维修后继续工作
3. 自动回家充电后继续工作
4. 自动凑出第一批 `Motor` 或 `BatteryPack`

### 4.7 设计重点

- 玩家应感受到自己不只是在写脚本，而是在搭建工作流程
- 据点应有明显“运转起来了”的反馈

### 4.8 完成标准

- 玩家第一次看到一条完整的“采集 -> 归还 -> 处理 -> 再出门”链条
- 玩家愿意开始自己组织代码结构

## 5. Milestone 4：危险垃圾区

### 5.1 体验目标

- 第一次明确让旧脚本失效

### 5.2 场景结构

- 普通垃圾区之外新增 1~2 种危险地块
- 示例：
  - 辐射废坑
  - 酸雨区
  - 旧电厂边缘

### 5.3 新增系统

- 辐射值
- 耐久更快下降
- 高价值危险区资源
- 更强的回撤需求

### 5.4 推荐新增资源

- `Chip`
- 稀有工业材料

### 5.5 玩家应学会的代码结构

- 风险判断
- 回撤逻辑
- 安全线 / 危险线
- “值不值得冒险”的优先级选择

### 5.6 推荐关卡任务

1. 从危险区带回第一个 `Chip`
2. 在耐久低于安全线时主动撤离
3. 在电量不足时放弃高风险目标
4. 稳定完成 5 次危险区回收而不报废

### 5.7 设计重点

- 不能只是更痛苦
- 必须是“高收益，但旧逻辑会死”
- 玩家应明显感受到重构带来的价值

### 5.8 完成标准

- “看到资源就捡”的脚本明显不够用了
- 玩家必须写判断逻辑才能稳定赚钱

## 6. M1~M4 机制清单

### 必须进入首版的机制

- 单机器人移动与转向
- 资源拾取与丢弃
- 基地
- 背包
- 电量
- 卸货
- 回充
- 维修
- 基础合成
- 危险地块
- 辐射 / 耐久判断

### 可以推迟到 M5 之后的机制

- 黑市价格波动
- 战斗 AI
- 敌对拾荒者
- 迷宫记忆
- 多机器人
- 排行榜

## 7. M1~M4 数值原则

### M1

- 几乎不设数值门槛
- 以动作反馈为主

### M2

- 掉落快
- 回充成本低
- 卸货收益可见

### M3

- 第一批配方短而清晰
- 设施成本不能太高
- 玩家能在短时间内看到据点循环成形

### M4

- 危险区收益必须显著高于普通区
- 但错误逻辑必须明显亏损
- 最好形成“稍微聪明一点就赚很多，继续无脑就会炸”的体验差

## 8. 建议的首版教学顺序

1. 手动体验机器人能动
2. 让玩家写出第一个顺序脚本
3. 教最简单的条件判断
4. 教循环
5. 教回家与卸货
6. 教回充与维修
7. 给第一条生产链
8. 给第一个危险规则
9. 用一张危险区关卡迫使重构

## 9. 验收问题

策划评审 M1~M4 时，统一问这几个问题：

1. 玩家在每个阶段到底学会了什么新编程问题？
2. 这一阶段新增的是“新能力”还是“新负担”？
3. 旧脚本具体为什么失效？
4. 新逻辑写出来之后，收益有没有明显提高？
5. 玩家有没有机会在低风险环境先练手？

## 10. 结论

Milestone 1~4 的任务不是把世界做大，而是把地基做对。

它们真正要建立的是：

- 机器人是有情感锚点的对象
- 代码能直接改造世界
- 自动化真的比手动有意义
- 新规则会逼迫玩家进化脚本

只要这四点立住，后面的黑市、战斗、迷宫、多机器人才会有承接。
