# Rust & Logic 资源分层与经济草表

本文是第一版资源与依赖链草案，目标不是锁死数值，而是先把资源层级、功能语义和依赖关系钉住。

## 1. 资源分层

### A. 基础资源

这些资源负责：

- 修理
- 基础建造
- 基础升级
- 低级模块制造

建议清单：

| 资源 | 说明 | 主要用途 | 出现阶段 |
| --- | --- | --- | --- |
| Scrap | 废铁、最基础的通货资源 | 修理、建造、基础配方 | M1 |
| Wire | 电线与连线材料 | 充电设施、基础电子配方 | M2 |
| Cell | 电池片 / 电芯碎片 | 电池与能源部件 | M2 |
| Plastic | 塑料壳体 | 外壳、轻型模块 | M2 |
| Oil | 润滑 / 燃料类低级工业材料 | 维修、移动模块、部分合成 | M3 |

### B. 工业资源

这些资源负责把基础资源转成据点和模块层能力。

| 资源 | 说明 | 主要用途 | 出现阶段 |
| --- | --- | --- | --- |
| Plate | 金属板材 | 装甲、结构件 | M3 |
| Motor | 基础电机 | 移动、机械臂、输送机构 | M3 |
| BatteryPack | 电池包 | 能源上限、续航模块 | M3 |
| SensorPart | 传感器组件 | 扫描、感知类模块 | M3-M4 |

### C. 技术资源

这些资源负责：

- 解锁新 API
- 推动剧情
- 打开高级功能

| 资源 | 说明 | 主要用途 | 出现阶段 |
| --- | --- | --- | --- |
| Chip | 芯片 | 模块升级、解锁逻辑能力 | M4 |
| Circuit | 电路板 | 高级模块、工坊升级 | M5 |
| SensorCore | 传感核心 | 高级扫描、迷宫感知 | M5-M7 |
| MemoryFragment | 记忆碎片 | 剧情推进、持久记忆能力 | M7 |
| OldCore | 旧世界核心碎片 | 终局设施与高阶集群能力 | M8+ |

### D. 战斗资源

这些资源负责角斗场与战斗线成长。

| 资源 | 说明 | 主要用途 | 出现阶段 |
| --- | --- | --- | --- |
| ArmorPlate | 装甲板 | 防御模块、维修消耗 | M6 |
| WeaponPart | 武器部件 | 武器模块、攻击升级 | M6 |
| Servo | 伺服部件 | 机动与战斗动作 | M6 |
| HeatSink | 散热器 | 持续战斗与高热管理 | M6 |
| Ammo | 弹药 | 远程武器与消耗循环 | M6 |
| ArenaToken | 角斗场凭证 / 战斗积分 | 高级战斗资格、战斗商店 | M6-M7 |

### E. 探索资源

这些资源负责迷宫、旧设施和剧情深度推进。

| 资源 | 说明 | 主要用途 | 出现阶段 |
| --- | --- | --- | --- |
| MapData | 地图数据 | 开图、导航升级、路径辅助 | M7 |
| KeyFragment | 钥匙碎片 | 门禁、设施权限 | M7 |
| Relic | 旧世界遗物 | 工艺、交易、剧情节点 | M7-M8 |
| Signal | 信号数据 | 设施定位、坐标解锁 | M7-M8 |
| FacilityPass | 设施通行权限 | 高阶区域访问 | M8 |

### F. 流动资源

这些不是普通库存，而是脚本判断条件。

| 资源 | 说明 | 设计用途 | 处理原则 |
| --- | --- | --- | --- |
| Energy | 当前电量 | 决定是否回充、继续工作 | 用危险线 / 安全线处理 |
| Heat | 当前热量 | 决定是否停火、降温 | 用节流，不追满 |
| Durability | 当前耐久 | 决定是否维修 / 回撤 | 用安全线处理 |
| Noise | 噪音值 | 决定是否吸引敌人 | 用环境风险处理 |
| Radiation | 辐射值 | 决定是否撤离危险区 | 用阈值 + 回撤逻辑处理 |

## 2. 资源依赖链

下面这张链路表的重点是：

- 高级目标不独立产出
- 每条高级线都反向依赖基础供给

### 2.1 基础据点链

```text
Scrap + Wire -> Motor
Cell + Wire -> BatteryPack
Scrap + Plastic -> Plate
Plate + Wire + Cell -> SensorPart
```

用途：

- 让玩家从“捡材料”进入“做部件”
- 让据点系统具备第一条真正意义上的生产链

### 2.2 危险区收益链

```text
基础修理 / 充电能力
-> 能稳定进入危险区
-> 危险区产出 Chip / 稀有工业材料
-> 芯片用于解锁新模块 / 新 API
```

用途：

- 把危险区和编程能力成长直接绑死
- 让“冒险”真正值钱

### 2.3 黑市与工坊经济链

```text
基础资源稳定供给
-> 可用于 trade / craft
-> 换取 Circuit / 蓝图 / 高级模块
-> 解锁更高价值生产目标
```

用途：

- 把固定刷资源推向动态补缺
- 让经济系统成为中期调度核心

### 2.4 战斗成长链

```text
基础工业资源
-> 造装甲 / 武器 / 散热 / 弹药
-> 打角斗场
-> 获得 ArenaToken / 高级零件
-> 购买更高战斗权限 / 模块
```

用途：

- 让战斗不是独立小游戏，而是资源系统中的一条主支线

### 2.5 迷宫探索链

```text
战斗资格 / 探索权限
-> 进入迷宫
-> 获得 MapData / KeyFragment / Relic / MemoryFragment
-> 解锁旧世界设施 / 高级 API / 剧情推进
```

用途：

- 让迷宫奖励和系统能力成长绑定
- 防止迷宫沦为简单宝箱系统

### 2.6 多机器人与终局链

```text
高级技术资源 + 剧情权限
-> 解锁集群协议
-> 修复更多机器人
-> 多线并发增长
-> 进一步放大供电、维修、调度压力
-> 进入聚落级自动化
```

用途：

- 把“多机器人”定义为复杂度跃迁，而不是简单数值跃迁

## 3. 隐性瓶颈设计

每条主线都应该有一个“表面目标”和一个“隐性瓶颈”。

| 表面目标 | 玩家以为缺什么 | 实际隐性瓶颈 |
| --- | --- | --- |
| 想升级扫描器 | Chip 不够 | 进入危险区的耐久与回撤逻辑不够 |
| 想造高级武器 | WeaponPart 不够 | 基础工业链与散热链不稳定 |
| 想刷迷宫奖励 | MapData / KeyFragment 不够 | 电量 / 战斗 / 路线管理不足 |
| 想多造机器人 | 资源总量不够 | 共享能源、维修吞吐、任务调度能力不足 |

这是本作中期和后期最重要的数值设计手段。

## 4. 流动资源处理规则

为了避免脚本抖动，以下资源统一采用“节流 / 滞回”原则：

### Energy

- `< 20%`：进入回充模式
- `> 80%`：退出回充模式

### Durability

- `< 30%`：回家维修
- `> 90%`：退出维修态

### Heat

- `> 危险线`：停止高热动作 / 拉开距离 / 切换低热动作
- `< 安全线`：恢复攻击节奏

### Noise

- 只在高风险区域作为额外判断条件
- 不做“追零”式处理

### Radiation

- 只在危险区工作时参与判断
- 超过阈值后触发回撤或换区

## 5. 首版数值设计建议

当前阶段不要急着设计精确公式，先锁这四条：

1. 基础资源掉落快，确保玩家能快速验证自动化收益
2. 工业资源合成链短，避免前期过度复杂
3. 芯片、战斗资源、迷宫资源明确分属不同玩法线
4. 流动资源优先服务脚本判断，而不是作为堆积目标

## 7. 结论

这份资源草表最重要的意义不是列名字，而是确立一件事：

> 资源系统必须服务于脚本重构，  
> 而不是服务于单纯刷数值。

只要这一点不丢，后续无论加多少材料和配方，系统都不会散。
