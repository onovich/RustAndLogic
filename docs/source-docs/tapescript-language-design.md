TapeScript 语言设计文档
1. 设计哲学与定位

TapeScript 是一种专为游戏《Rust & Logic》设计的极简、无状态、低抽象层级的领域特定语言（DSL）。 它的核心目标并非图灵完备或通用计算，而是通过语言本身的物理限制，提供硬核且迷人的资源调度体验。

没有嵌套，单向流动： 摒弃括号和作用域，代码自上而下运行。

物理隐喻： 代码被视为实体的“打孔纸带”。代码行数受限于机器人的物理内存容量。

无形状态： 语言层面没有变量定义，状态存储在机器人的硬件模块（如条件寄存器、装甲值传感器）中。

2. 核心语法规则

执行顺序： 严格按行逐次执行。

单行单指令： 每一行只能包含一个标签或一条有效指令。

注释： 使用 // 进行人类备注，编译时将被剔除，不占用纸带容量。

大小写敏感： 指令和标签区分大小写，以确保解析的绝对精确。

3. 语言结构
3.1 标签 (Label)

跳转的锚点。必须以 @ 符号开头，独占一行。

示例：@MainLoop

3.2 动作指令 (Action)

调用机器人底层的物理或电子模块，直接改变世界状态。

示例：MoveForward, TurnLeft, PickUp, Fire

3.3 传感器查询 (Query)

查询世界或自身状态，结果隐式存入实体的“条件寄存器 (Condition Flag)”。

示例：CheckScrap（前方是否有废料？）, CheckHP_Low（血量报警？）

3.4 控制流 (Control Flow)

基于条件寄存器的状态，改变程序计数器（PC）的位置。

示例：Jump @MainLoop, JumpIfTrue @Attack

4. 纸带容量（内存限制机制）

容量即数值： 机器人的等级决定了“纸带长度”（例如初级只有 8 行）。

指令消耗： 每一条动作、查询、跳转指令消耗 1 行物理纸带。

标签消耗： 标签 @ 同样消耗 1 行纸带。这迫使玩家在优化代码时，尽量减少不必要的逻辑跳转。
